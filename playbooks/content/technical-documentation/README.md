# Technical Documentation

## Use When

Use this playbook when technical facts or workflows need to become a
reader-focused document rather than a loose internal note.

## Inputs

- documentation goal
- target reader
- source material context

## Output

The workflow should produce:
- target reader and document goal
- drafted documentation
- important limits
- editorial review findings
- follow-up documentation gaps

## Workspace Notes

This playbook works with a writable documentation workspace or in
advisory mode. A writable workspace improves the handoff but is not
required for honest completion.

## Import Notes

This playbook is strongest when the imported specialists can access the
same source truth the human author would use.
