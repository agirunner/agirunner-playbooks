# SOP Creation

## Use When

Use this playbook when a repeated internal process needs to become a
clear standard operating procedure instead of tribal knowledge.

## Inputs

- procedure goal
- current process context
- target audience
- any existing draft material or constraints

## Output

The workflow should produce a usable SOP draft with:
- scope and trigger
- owner
- inputs
- procedure steps
- exception handling
- completion definition

## Workspace Notes

This playbook works with writable workspaces and with artifact-only
inputs. In advisory contexts it should still produce a complete SOP
draft in the handoff.

## Import Notes

This playbook works best when the imported specialists can access source
policy, existing process notes, and any currently used operator guides.
