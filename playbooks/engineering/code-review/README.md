# Code Review

## Use When

Use this playbook when a proposed code or configuration change needs a
clear review disposition before merge, adoption, or release planning.

## Inputs

- review scope
- change goal
- available evidence if any

## Output

The workflow should produce:
- primary review findings
- security findings
- verification gaps
- final review recommendation

## Workspace Notes

This playbook is storage-agnostic and can complete from change artifacts
alone.

## Import Notes

This playbook is experimental and intended for bounded review workflows,
not as a substitute for larger delivery governance.
