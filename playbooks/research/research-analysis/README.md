# Research Analysis

## Use When

Use this playbook when a team needs a grounded answer to a specific
research question, along with source-based findings and honest
confidence.

## Inputs

- research question
- decision context if relevant
- source scope or source constraints if relevant

## Output

The workflow should produce:
- research question framing
- sources consulted
- findings
- contradictions or gaps
- confidence
- recommendation or next step

## Workspace Notes

This playbook can operate entirely from artifacts and external research
access. It does not require a writable code workspace.

## Import Notes

Best results come when the imported research specialist has access to a
good web research provider or equivalent MCP integration, but the
workflow should still function with documented source limits if that is
not configured.
